Imagens em HTML

Em HTML, imagens são usadas para ilustrar conteúdos, explicar ideias e deixar a página mais atrativa. Para inserir imagens utilizamos a tag <img>, que é autofechante (não possui tag de fechamento). Os atributos essenciais são: src (fonte da imagem) e alt (descrição textual para acessibilidade e fallback).

Estrutura Básica da Imagem (<img>)

A forma mais simples de inserir uma imagem é informando o src com o caminho do arquivo e o alt com uma descrição curta e objetiva do que ela mostra. 

<img src="viralata-caramelo.webp" alt="Um vira-lata caramelo deitado no pátio">

Imagens Locais e Externas

Você pode usar imagens salvas no seu projeto (caminho relativo) ou imagens hospedadas na web (caminho absoluto com URL). Lembre-se de manter nomes de arquivos coerentes e pastas organizadas. 

<!-- Imagem local, dentro da pasta do projeto -->
<img src="assets/gatinho.webp" alt="Gatinho deitado olhando para a câmera">

<!-- Imagem externa, hospedada em outro servidor -->
<img src="https://stock.adobe.com/gatinho-deitado.webp" alt=Gatinho deitado">

Importante!
Quando utilizamos imagens externas em nossos projetos, devemos lembrar que a correta exibição das imagens em nosso site depende do perfeito funcionamento do site onde as imagens estão hospedadas e, caso haja alguma indisponibilidades por parte do site que hospeda as imagens, elas não serão exibidas em nosso site.

Tamanho da Imagem (width e height)

Os atributos width e height redimensionam a imagem. Informe apenas um deles para manter a proporção automaticamente. Se informar ambos com valores que não correspondem à proporção original, a imagem pode distorcer. 

<!-- Largura fixa, altura proporcional -->
<img src="gato.jpg" alt="Gato tigrado" width="300">

<!-- Largura e altura definidas (cuidado com distorções) -->
<img src="gato.jpg" alt="Gato tigrado" width="300" height="200">

Título da Imagem (title)

O atributo title adiciona uma dica visual (tooltip) ao passar o mouse sobre a imagem. Ele não substitui o alt, que é obrigatório para acessibilidade. 

<img src="camera.png" alt="Ícone de câmera fotográfica" title="Clique para ver a galeria">

Quando a Imagem Não Carrega

Se o arquivo não existir ou a conexão falhar, o navegador tenta exibir o alt. Por isso, mantenha esse texto sempre descritivo e útil. 

<!-- Caminho propositalmente incorreto para demonstrar o alt -->
<img src="nao-existe/foto.png" alt="Retrato de uma pessoa sorrindo">

Formatos de Imagens

Os arquivos de imagens são importantes recursos para nossas páginas da web, entretanto, não são todos os formatos de arquivo que têm suporte para web no navegadores atuais.

Na web e no design, escolher o formato certo impacta qualidade, tamanho do arquivo, compatibilidade e recursos como transparência e animação.

Abaixo, apresentamos uma relação dos principais tipos de arquivos de imagens:

JPG (JPEG)

História: criado em 1992 pelo Joint Photographic Experts Group, popularizou-se para fotografias digitais por oferecer alta compressão.

Vantagens:

  • Alta taxa de compressão com boa qualidade perceptiva.
  • Compatibilidade praticamente universal (web, apps, sistemas).
  • Ideal para fotos e imagens com muitos tons e gradientes.

Desvantagens:

  • Compressão com perdas (cada regravação pode degradar).
  • Sem suporte a transparência.
  • Não é ideal para textos, logos e gráficos chapados.

Quando usar: fotografias e banners fotográficos quando a compatibilidade é prioridade.

PNG

História: lançado em 1996 como alternativa aberta ao GIF, focado em qualidade sem perdas e transparência.

Vantagens:

  • Compressão sem perdas, preservando detalhes.
  • Suporta transparência real (canal alpha) e bordas suaves.
  • Ótimo para UI, ícones, logos e gráficos com áreas chapadas.

Desvantagens:

  • Arquivos maiores que JPG para fotos.
  • Menos eficiente para imagens fotográficas grandes.

Quando usar: elementos de interface, logotipos, prints e imagens que exijam transparência nítida.

GIF

História: de 1987 (CompuServe), tornou-se ícone da web por suportar animação em conexão lenta.

Vantagens:

  • Compatibilidade ampla em navegadores e mensageiros.
  • Animações simples em um único arquivo.
  • Bom para gráficos com paleta reduzida.

Desvantagens:

  • Limitado a 256 cores (ruim para fotos e gradientes).
  • Sem transparência parcial (apenas tudo ou nada).
  • Arquivos geralmente maiores que vídeos curtos/anim. modernas.

Quando usar: apenas por compatibilidade/nostalgia; prefira WebP animado ou vídeo (MP4/WebM) para eficiência.

SVG

História: padrão do W3C (1999) para gráficos vetoriais escaláveis e interativos.

Vantagens:

  • Escala infinita sem perda de qualidade (vetorial).
  • Arquivos leves, editáveis (XML), estilização via CSS.
  • Suporta interatividade e animações com CSS/JS.

Desvantagens:

  • Não serve para fotografias (raster).
  • SVGs complexos podem pesar/ser trabalhosos de renderizar.
  • Exige atenção a segurança (scripts embutidos quando externo).

Quando usar: logos, ícones, ilustrações, gráficos e mapas interativos.

WebP

História: criado pelo Google (2010) para reduzir tamanho de imagens na web com boa qualidade.

Vantagens:

  • Compressão com e sem perdas; geralmente menor que JPG/PNG.
  • Suporta transparência e animação (substitui muitos GIFs).
  • Qualidade visual competitiva com tamanhos reduzidos.

Desvantagens:

  • Geração/decodificação pode ser mais pesada que JPG/PNG.
  • Compatibilidade antiga/legada ainda pode ser um obstáculo específico.

Quando usar: web moderna quando se busca melhor performance sem abrir mão de transparência/anim.

AVIF

História: baseado no codec AV1 (2018); projetado para compressão de última geração.

Vantagens:

  • Taxa de compressão superior (arquivos menores com alta qualidade).
  • Suporta HDR, transparência e animações.
  • Excelente para otimização de performance em sites e apps.

Desvantagens:

  • Codificação/decodificação mais custosa (CPU).
  • Compatibilidade ainda não é universal em ferramentas/ambientes legados.

Quando usar: otimização máxima de imagens (especialmente fotos) em projetos modernos com suporte adequado.

Qual usar em cada situação?

  • Fotos: AVIF (melhor), WebP (ótimo), JPG (compatibilidade universal).
  • Logos/Ícones/Ilustrações: SVG (preferencial). PNG quando precisar raster com transparência.
  • Transparência: PNG (clássico), WebP/AVIF (modernos), SVG (vetor).
  • Animação: WebP animado ou vídeo curto (MP4/WebM); GIF só por compatibilidade.
  • Compatibilidade máxima sem surpresas: JPG/PNG/SVG.
  • Performance web moderna: AVIF > WebP > JPG/PNG (como fallback).

Resumo

- Use <img> para inserir imagens na página.
- Defina src (caminho da imagem) e alt (descrição acessível).
- Redimensione com width e/ou height (evite distorções).
- Use title para dica ao passar o mouse (opcional).
- Para responsividade simples, utilize width="100%" no <img> dentro de um contêiner.
- Prefira imagens otimizadas em formatos adequados (JPG, PNG, GIF, SVG, WebP) e nomes de arquivos claros.
