CSRF (Cross-site Request Forgery)

É um ataque cibernético que engana um navegador para que ele execute uma ação indesejada em uma aplicação web onde o usuário já está autenticado, explorando a confiança da aplicação no usuário para realizar ações que ele não pretendia, como fazer uma transferência de fundos ou alterar uma senha.

Como funciona um ataque CSRF

  1. Autenticação:

    2. O usuário acessa um site e faz login, e seu navegador fica autenticado nesse site, mantendo um cookie de sessão ativo.

  2. Engenharia Social:

    3. O invasor cria uma página maliciosa (por exemplo, em um e-mail ou link), que, ao ser acessada, faz o navegador do usuário enviar uma solicitação para o site onde ele está autenticado.

  3. Execução da Ação:

    4. O site recebe a solicitação, que parece ser legítima porque contém o cookie de sessão do usuário. O site, então, executa a ação (como alterar uma senha, transferir dinheiro, etc.) como se o usuário a tivesse solicitado voluntariamente

Impacto de um ataque CSRF

Um ataque bem-sucedido pode ter consequências graves, incluindo: Transferências de fundos não autorizadas, Alteração de senhas e configurações da conta, Roubo de dados da conta do usuário, Prejuízos à reputação da empresa.

Como se proteger?

Para se defender contra ataques CSRF, desenvolvedores de aplicações web utilizam técnicas como a implementação de tokens CSRF, que são gerados para cada sessão e são verificados antes de qualquer ação ser executada.
