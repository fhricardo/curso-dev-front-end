Formulários em HTML

Formulários permitem coletar informações dos usuários (texto, escolhas, arquivos, etc.). A tag principal é <form>, que organiza os campos e define action (para onde os dados vão) e method (get ou post). Importante: o formulário coleta os dados; para processar (salvar, enviar e-mail, autenticar) você normalmente precisa de um back-end (PHP, Node, Python, etc.).

Estrutura básica do formulário (<form>)

Os atributos essenciais do <form> são:

  • action — destino (URL/arquivo) que receberá os dados.
  • method — get ou post.
  • enctype — necessário para upload de arquivos: multipart/form-data.
<form action="processa.php" method="post" enctype="multipart/form-data">
    <label for="nome">Nome:</label>
    <input type="text" id="nome" name="nome">

    <input type="submit" value="Enviar">
</form>

Métodos de envio: GET vs POST

Entenda as diferenças para escolher corretamente:

  • GET — envia dados na URL (ex.: ?q=termo). Bom para buscas e filtros. Funciona só com HTML (você vê os parâmetros na URL). Tem limite de tamanho e não é seguro para dados sensíveis.
  • POST — envia dados no corpo da requisição. Uso recomendado para cadastros, logins e uploads. Para proces­sar os dados recebidos via POST você precisa de back-end.
<!-- Exemplo GET -->
<form action="/buscar" method="get">
  <label for="q">Buscar:</label>
  <input type="text" id="q" name="q">
  <input type="submit" value="Pesquisar">
</form>

<!-- Exemplo POST (requere backend para processar) -->
<form action="/cadastro" method="post">
  <label for="user">Usuário:</label>
  <input type="text" id="user" name="user">
  <input type="submit" value="Cadastrar">
</form>

É importante lembrar que o método post exige a utilização de uma linguagem back-end (como o PHP, por exemplo) para que as informações sejam processadas.

Tipos de campos e exemplos práticos

Cada tipo tem seu propósito — use o mais adequado para UX/validação:

  • text, password, email, tel, url
  • number, range
  • date, time, datetime-local
  • radio (escolha única) e checkbox (múltipla)
  • file (upload — require enctype)
  • textarea (texto longo) e select (dropdown)
<form action="#" method="post" enctype="multipart/form-data">

  <!-- texto, senha e email -->
  <label for="nome">Nome:</label>
  <input type="text" id="nome" name="nome" maxlength="100" size="30">

  <label for="senha">Senha:</label>
  <input type="password" id="senha" name="senha" maxlength="30">

  <label for="email">E-mail:</label>
  <input type="email" id="email" name="email">

  <!-- number e date -->
  <label for="idade">Idade:</label>
  <input type="number" id="idade" name="idade" min="0" max="120">

  <label for="nasc">Nascimento:</label>
  <input type="date" id="nasc" name="nasc">

  <!-- file (upload) -->
  <label for="foto">Foto:</label>
  <input type="file" id="foto" name="foto">

  <!-- textarea -->
  <label for="msg">Mensagem:</label>
  <textarea id="msg" name="msg" rows="4" cols="40"></textarea>

  <input type="submit" value="Enviar">
</form>








Uso correto de radio e checkbox (labels e agrupamento)

Para usabilidade e acessibilidade: associe cada input a um <label for="id">. Agrupe radios/checkboxes relacionados com <fieldset> e descreva com <legend>. 

<fieldset>
  <legend>Gênero</legend>

  <input type="radio" id="g-m" name="genero" value="M">
  <label for="g-m">Masculino</label>

  <input type="radio" id="g-f" name="genero" value="F">
  <label for="g-f">Feminino</label>

  <input type="radio" id="g-o" name="genero" value="O">
  <label for="g-o">Outro</label>
</fieldset>

<fieldset>
  <legend>Interesses</legend>
  <input type="checkbox" id="i-html" name="interesse[]" value="html">
  <label for="i-html">HTML</label>

  <input type="checkbox" id="i-css" name="interesse[]" value="css">
  <label for="i-css">CSS</label>
</fieldset>

Botões em Formulários

Elementos fundamentais em formulários, os botões permitem a interatividade dos usuários e a execução de tarefas, como o próprio envio do formulário.

Os botões podem ser inseridos em formulários das seguintes formas:

No formato de inputs:

  • type="submit": o submit é quem executa o envio do formuário e deve existir apenas uma vez em um form.
  • type="reset": o reset é responsável por limpar o conteúdo de um form e também deve ser utilizado apenas uma vez em cada form.
  • type="button": o tipo button não possui uma ação definida, sendo necessário utilizar JavaScript para atribuir funções específicas a eles. Podem ser utilizados diversos do tipo button em um form.
...
<!-- No formato de input -->
<input type="reset" value="Limpar">
<input type="submit" value="Enviar">
<input type="button" value="Botão">
...

No formato de buttons:

Com uma estrutura um pouco diferente dos inputs que mostram botões, a tag <button> requer a tag de fechamento <button> e pode ter os seguintes tipos:

  • type:"submit": Assim como no input, o button com o type submit tem como função o envio do formulário.
  • type:"reset": Também como acontece no input, o type reset faz com que a função do button seja a de limpar o formulário.
  • type:"button": o button do tipo button é o nosso botão genérico, utilizado quando vamos atribuir funções utilizando JavaScript.
...
<!-- No formato de input -->
<button type="submit">Enviar</button>
<button type="reset">Limpar</button>
<button type="button">Botão</button>
...

É importante lembrar que o <button> sem o atributo type definido, assume, por padrão, a função de submit e vai enviar o formulário.

Utilizar <button type="button">Botão</button> ao invés de <input type="button" value="Botão"> é mais interessante do ponto de vista de personalização, uma vez que nos permite atribuir ícones aos botões com facilidade, por meio da utilização de bibliotecas de ícones como fontawesome

Validação nativa do HTML e atributos úteis

HTML tem validação básica de campos sem JS (ótimo pra UX). Alguns atributos importantes:

  • required — campo obrigatório.
  • pattern — regex para validar o valor (ex.: telefones, CPFs simples).
  • min / max — para number e datas.
  • maxlength / minlength — limite de caracteres permitido.
  • placeholder — dica dentro do campo (não substitui label).
  • autocomplete — controla preenchimento automático.
<input type="text" name="usuario" required minlength="3" maxlength="20" placeholder="seu_nome">

<input type="tel" name="fone" pattern="^\(\d{2}\)\s?\d{4,5}-\d{4}$" placeholder="(11) 99999-9999">

Quando você precisa de back-end

HTML cuida da coleta e validação básica no cliente. As ações abaixo exigem processamento no servidor:

  • Salvar dados em um banco (MySQL, PostgreSQL, MongoDB...)
  • Enviar e-mail com informações do formulário
  • Autenticar usuários (login/senha)
  • Tratar uploads (salvar arquivos, validar tamanho/tipo)

Em resumo: <form method="get"> pode ser testado só com HTML (você verá parâmetros na URL). Já post normalmente precisa de um endpoint no back-end para realmente fazer algo útil.

Boas práticas e acessibilidade

  • Use sempre <label for="..."/> — amplia área clicável e ajuda leitores de tela.
  • Agrupe campos relacionados com <fieldset> e descreva com <legend>.
  • Forneça mensagens de erro claras (validação no front com JS e no back-end).
  • Use atributos ARIA apenas quando necessário; prefira HTML semântico primeiro.
  • Proteja formulários contra CSRF e valide tudo no servidor (não confie na validação do cliente).

Exemplo completo: Formulário de cadastro

<form action="/cadastro" method="post" enctype="multipart/form-data">
  <label for="nome">Nome completo</label>
  <input type="text" id="nome" name="nome" required maxlength="100">

  <label for="email">E-mail</label>
  <input type="email" id="email" name="email" required>

  <fieldset>
    <legend>Gênero</legend>
    <input type="radio" id="m" name="genero" value="M">
    <label for="m">Masculino</label>

    <input type="radio" id="f" name="genero" value="F">
    <label for="f">Feminino</label>

    <input type="radio" id="o" name="genero" value="O">
    <label for="o">Outro</label>
  </fieldset>

  <label for="curso">Curso</label>
  <select id="curso" name="curso">
    <option value="frontend">Front-End</option>
    <option value="backend">Back-End</option>
    <option value="design">Design</option>
  </select>

  <label for="foto">Foto (opcional)</label>
  <input type="file" id="foto" name="foto">

  <input type="submit" value="Cadastrar">
</form>

Resumo

- Sempre associe <label for> aos inputs.
- Use fieldset/legend para agrupar opções (radios/checkboxes).
- GET exibe parâmetros na URL; POST envia no corpo (requer back-end para processar).
- Para upload use enctype="multipart/form-data".
- Valide no cliente (HTML/JS) e no servidor — nunca confie só no cliente.
- Proteja seus endpoints (CSRF, validação, sanitização).
