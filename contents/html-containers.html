Containers em HTML

Containers são elementos utilizados para agrupar e organizar outros elementos dentro da página. Eles não possuem, por si só, um significado semântico (com exceção dos containers semânticos que veremos depois), mas ajudam na estruturação e no estilo com CSS.

Principais Containers Genéricos

Antes do HTML5, praticamente todo o conteúdo era organizado usando apenas dois elementos principais:

  • <div> – Container em nível de bloco. Usado para agrupar seções maiores, como menus, áreas de conteúdo ou rodapés.
  • <span> – Container em nível de linha. Usado para destacar ou estilizar trechos de texto sem quebrar a linha.

Outros Containers Úteis

  • <iframe> – Cria um “quadro embutido” que carrega outra página dentro do documento atual.
  • <fieldset> – Agrupa campos de um formulário.
  • <details> – Cria um container que pode ser expandido ou recolhido pelo usuário.

Exemplo de Containers em Ação

<div class="conteudo">
    <h2>Título de uma seção</h2>
    <p>Esse parágrafo está dentro de um container div.</p>

    <p>Podemos usar <span class="destaque">span</span> 
       para destacar um trecho de texto.</p>

    <iframe src="https://www.example.com" width="300" height="200"></iframe>
</div>

Resumo

- Use <div> para agrupar blocos de conteúdo.
- Use <span> para destacar partes inline de um texto.
- <iframe> permite inserir páginas externas.
- <fieldset> organiza campos de formulários.
- <details> cria containers que podem ser abertos ou fechados.
- Containers são a base da estrutura, e com o HTML5 eles evoluem para as tags semânticas, que veremos na próxima aula.
